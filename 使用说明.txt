这个 Agent 本质是一套标准化工作指令集，不需要额外安装插件或工具，你只需要把需求 + 本规范文档发给 AI（也就是当前对话），AI 就会严格按照这套规则输出对应产物。下面是三种最常用的调用方式，附带可直接复制的指令模板。

一、核心调用逻辑
你只需要在提问时明确两件事：
你的项目基础信息（品类、引擎、平台、规模等级、你的开发基础）
你想要的产出（全套指导书 / 单个章节 / 专项文档 / 代码模板）
AI 会自动匹配规范里的分级规则、跨引擎模板、零基础引导、商业化配置等能力，输出符合标准的结果。

二、三种常用调用方式（带可复制模板）
1. 【全量触发】从零生成完整项目指导书
适用场景：只有一个游戏想法，从 0 到 1 输出全套设计文档 + 分章指导书 + 开发计划。
标准指令模板：
plaintext
请严格遵循《游戏项目指导书生成 Agent 工作规范》执行。
项目信息：
游戏品类：xxx
核心玩法/乐趣点：xxx
目标平台：xxx
开发引擎：xxx
项目等级：L1/L2/L3/L4 选一个
我的开发基础：零基础/入门/有经验
是否需要商业化：是/否

输出要求：
1. 先提取项目核心信息，识别风险点并给出常见坑预警
2. 按对应等级生成全套基础文档（gdd/system_design/art_direction/plan）
3. 生成完整的分章项目指导书，每个 Task 按标准模板输出
4. 开发计划附带甘特图和周度时间轴
5. 零基础请附带专属入门引导包
示例（可直接修改使用）：
请严格遵循《游戏项目指导书生成 Agent 工作规范》执行。
项目信息：
游戏品类：合成类庄园小游戏
核心玩法 / 乐趣点：拖拽合成植物，解锁新地块，接订单赚金币
目标平台：微信小游戏
开发引擎：Cocos Creator 2D
项目等级：L2 标准 MVP 级
我的开发基础：零基础
是否需要商业化：是，激励视频为主
2. 【阶段触发】推进现有项目的单个章节 / 任务
适用场景：已经有项目文档和指导书，现在要推进某一个具体章节 / 单个功能，对应规范第 17 节的标准开场。
标准指令模板：
plaintext
请先阅读并遵循《游戏项目指导书生成 Agent 工作规范》，以及项目已有文档：
docs/gdd.md
docs/system_design.md
docs/art_direction.md
docs/plan.md
docs/{游戏名}项目指导书/{当前章节}.md

当前目标：
[写清楚单个可验证的具体任务，比如“实现04章核心机制里的拖拽合成功能”]

约束：
1. 先说明计划修改的文件与整体思路。
2. 只实现当前目标，不额外扩展无关功能。
3. 遵循项目统一的代码规范与命名规则。
4. 完成后说明具体的验证操作步骤。
5. 如需引擎内操作，写清点击路径。
6. 验证通过后给出建议的存档备注。

我的开发基础：零基础/入门/有经验
示例：
请先遵循 Agent 工作规范，当前项目是合成庄园小游戏，已完成初始化。
当前目标：实现 04 核心机制章节的「拖拽物品合并升级」功能，支持两个相同等级物品拖拽到一起自动升级，产出金币。
约束：用 Cocos TypeScript 实现，附带完整代码片段和验证步骤。
我的开发基础：入门级
3. 【专项触发】只生成某一类专项产物
适用场景：不需要全套指导书，只需要某个独立模块的成果，比如埋点代码、审核清单、商业化配置等。
① 生成可直接用的数据埋点代码
plaintext
请按 Agent 规范，生成一套 [Cocos/Unity/Godot] 引擎的小游戏数据埋点代码模板，要求：
1. 支持微信/抖音小游戏平台上报
2. 包含启动、关卡开始/结束、广告展示/点击、付费这几个核心事件
3. 附带统一的事件命名规范和参数说明
4. 可直接复制到项目里使用
② 生成小游戏上线审核自动检查清单
plaintext
请按 Agent 规范里的合规要求，生成一份 [微信/抖音] 小游戏上线审核检查清单，要求：
1. 区分必过项和可选项
2. 每项标注检查方法和驳回风险
3. 附带常见驳回原因和修改方案
4. 按功能、合规、素材、包体分类
③ 生成商业化配置化文件

请按 Agent 规范的商业化配置方案，生成一套合成类小游戏的广告位+内购配置文件（JSON格式），要求：
1. 包含复活、双倍收益、补充体力3个激励视频位
2. 包含3档金币内购商品
3. 带每日次数限制、冷却时间、开关字段
4. 附带配置说明和修改方法
④ 生成项目甘特图 / 时间轴
plaintext
请按 Agent 规范，为 [项目名称] 生成一份开发甘特图和周度时间轴，要求：
1. 按 L2 MVP 等级，周期2周
2. 用 Mermaid 格式输出甘特图
3. 附带周度任务表格，标注前置依赖和交付物
4. 按“骨架→核心玩法→内容→UI→打磨→交付”的顺序排期
三、调用注意事项
信息越全，结果越准
尽量一次性说清品类、引擎、平台、等级，不要只说 “帮我做个游戏”，避免反复确认。不确定的项可以不说，AI 会按默认值处理并标注。
严格遵循单目标原则
一次只提一个可验证的任务，比如 “先做核心拖拽合成”，不要一次性要求 “把合成、订单、广告、UI 全做了”，否则会影响质量和可验证性。
报错反馈按规范格式
遇到功能异常时，按「操作步骤→预期结果→实际现象→完整日志」四要素反馈，AI 会按规范里的排障流程定位修复。
零基础追加一句话即可
任何指令最后加一句 “我是零基础，请附带详细操作步骤和完整可复制的代码”，就会自动启用零基础引导模式。
引擎切换直接说明
明确说 “用 Unity 实现”“用 Godot 实现”，AI 会自动加载对应引擎的目录结构、代码模板和最佳实践。
